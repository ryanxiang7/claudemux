add .workspace/ for dispatcher personalization and intermediate artifacts

`/claudemux:setup` now seeds `<dispatcher-dir>/.workspace/` with:

- `profile/persona.md`, `profile/user-profile.md`, `profile/principles.md` — HTML-comment-only stubs you fill in; imported into every dispatcher session via `.workspace/imports.md` (chain: `CLAUDE.md` → `.workspace/imports.md` → three `profile/*.md`).
- `imports.md` — explains the layout and `@import`s the three profile files.
- `notes/` — long-term human-curated local notes; not auto-imported.
- `artifacts/<YYYYMMDD>-<slug>.md` — dispatcher-generated intermediate output; replaces ad-hoc `/tmp/<topic>.md`; not auto-imported.
- `.workspace/.git/` — `.workspace/` is initialized as its own independent git repo with an initial commit on first setup. Completely separate from any git tracking at the dispatcher-dir level.

`templates/CLAUDE.md.template` gains a marker-bracketed `@.workspace/imports.md` line at the top (between `<!-- claudemux-workspace-imports:start --> ... :end -->`). Fresh installs get this automatically.

`--force` semantics narrowed: previously `--force` overwrote the dispatcher's CLAUDE.md from the template. Now `--force` on a customized CLAUDE.md only injects the marker block at the top (preserving the body). This is strictly safer — the prior behavior could clobber user edits.

`/claudemux:setup` slash command gains a Step 1.5 that walks the user through the three profile files (text-first; the modal `AskUserQuestion` form is reserved for clients that surface it, since Remote Control web and the mobile app do not). Skipping leaves the HTML-comment stubs in place (no behavioral difference, only no personalization yet). The walk-through is gated on the `workspace_imports_active` status `setup.sh` emits — a customized legacy `CLAUDE.md` without the marker block ends up in the "stubs seeded but inactive" state, which the slash command surfaces to the user along with the `--force` recovery hint instead of silently asking them to fill stubs that have no effect.

Defensive guards in `setup.sh`: refuses to seed scaffold or `git init` when `<dispatcher-dir>/.workspace/` is a symlink (would place `.git/` outside the dispatcher dir, violating the per-dispatcher-archive contract); captures git's stderr on commit failure and surfaces it as ` git: ...` diagnostic lines (graceful fall-through, no script exit); `${VAR:?}` guards on `rm -rf` paths in the conformance test (ShellCheck SC2115). CI invokes the conformance test at `plugins/claudemux/scripts/test-setup.sh` (8 scenarios, 32 assertions) on every push.

Positioned as **standalone Phase 0 / building block of #85**: the workspace covers single-dispatcher personalization and ad-hoc artifact parking. It explicitly does not touch the task ledger (stays in AutoMemory), teammate ownership, `tm states` fleet scope, cron host rules, or any other multi-workspace isolation question. Those remain open for #85; today's layout is forward-compatible with two migration paths (move contents into a `default/` subdir, or rename top-level to `.workspaces/`).

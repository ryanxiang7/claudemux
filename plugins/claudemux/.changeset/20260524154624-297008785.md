Breaking exit-code split: `tm send` / `tm wait` / `tm spawn --prompt` / `tm compact` now exit `124` (GNU `timeout(1)` convention) when their `--timeout` elapses without a Stop / PostCompact hook (was `1`), while true failures (no session, no sid, sendKeys dead) keep `1`. The wait-expiry path also re-probes the teammate before returning `124` so a session that died mid-wait surfaces as `1` instead of being mis-reported as "still running". Codex sync-wait expiries align on `124` and `tm wait` now uses a `thread/read` backfill alongside the live subscription, surfacing any TERMINAL turn (completed / failed / interrupted) that resolved in the gap between a previous `tm send` timing out and the new wait subscribing — so late failures map to exit `1` instead of spinning to `124`, identical to the live-notification path.
